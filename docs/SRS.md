## ۱. مقدمه

### ۱.۱ هدف پروژه

هدف از این پروژه، ساخت یک اپلیکیشن دسکتاپ ساده برای مدیریت یک فروشگاه مواد غذایی است. این برنامه با هدف یادگیری کار با کتابخانه PyQT و آشنایی با مفاهیم پایه توسعه اپلیکیشن‌های دسکتاپ با پایتون طراحی شده است.

### ۱.۲ مخاطبان

- توسعه‌دهندگان مبتدی پایتون
    
- دانشجویانی که می‌خواهند با PyQT آشنا شوند
    
- افرادی که به دنبال یک پروژه نمونه برای یادگیری هستند
    

### ۱.۳ دامنه پروژه

این پروژه یک نسخه دمو و ساده از یک سیستم فروشگاهی است که قابلیت‌های پایه‌ای مانند مدیریت محصولات، ثبت سفارش و ذخیره‌سازی داده‌ها را پیاده‌سازی می‌کند.

---

## ۲. شرح کلی سیستم

### ۲.۱ دید کلی

این برنامه یک واسط کاربری گرافیکی (GUI) با استفاده از PyQT ارائه می‌دهد که کاربران می‌توانند از طریق آن محصولات را مشاهده کنند، به سبد خرید اضافه کنند و سفارش خود را ثبت نمایند.

### ۲.۲ کاربران سیستم

- **مدیر فروشگاه**: می‌تواند محصولات را مدیریت کند
    
- **کاربر عادی**: می‌تواند سفارش ثبت کند
    

### ۲.۳ معماری سیستم

معماری این پروژه شامل سه لایه اصلی است: لایه نمایش (رابط کاربری PyQT)، لایه منطق (مدیریت محصولات و سبد خرید) و لایه داده (ذخیره‌سازی در فایل‌های JSON). توضیحات کامل همراه با دیاگرام‌های معماری، کلاس‌ها، توالی، فعالیت و استقرار در فایل `docs/architecture.md` موجود است.

## ۵. معماری و طراحی

### ۵.۱ ساختار پوشه‌ها

```bash
food-shop-demo/
├── app/                    # ماژول‌های اصلی برنامه
│   ├── controllers/        # منطق کسب‌وکار
│   ├── models/             # مدل‌های داده
│   └── views/              # کامپوننت‌های UI
├── docs/                   # مستندات
│   ├── architecture.md     # دیاگرام‌های معماری
│   └── srs.md              # این فایل
├── media/                  # تصاویر و آیکون‌ها
├── reports/                # گزارشات تولید شده
├── tests/                  # تست‌های واحد
├── main.py                 # نقطه ورود برنامه
├── profile.json            # تنظیمات کاربر
└── requirements.txt        # وابستگی‌ها
```


### ۵.۲ معماری لایه‌ها

برنامه از سه لایه اصلی تشکیل شده است:

1. **لایه نمایش (Presentation Layer)**: شامل کلاس `MainWindow` و کامپوننت‌های PyQT که مسئول نمایش اطلاعات و دریافت ورودی از کاربر هستند.
    
2. **لایه منطق (Business Logic Layer)**: شامل کلاس‌های `Product`، `CartItem` و `Order` که منطق اصلی برنامه را پیاده‌سازی می‌کنند.
    
3. **لایه داده (Data Layer)**: شامل کلاس `DataManager` که مسئول ذخیره‌سازی و بازیابی اطلاعات از فایل‌های JSON است.
    

برای مشاهده دیاگرام‌های دقیق معماری، کلاس‌ها، توالی و فعالیت به فایل `docs/architecture.md` مراجعه کنید.

## ۶. مراحل توسعه

### مرحله ۱: راه‌اندازی اولیه

**قدم‌ها:**

1. نصب پایتون ۳.۸ یا بالاتر
    
2. نصب PyQT: `pip install pyqt5`
    
3. ساخت ساختار پوشه‌ها
    
4. ایجاد فایل `main.py` به عنوان نقطه ورود
    

**خروجی:** یک پنجره خالی PyQT که به درستی اجرا می‌شود.

### مرحله ۲: طراحی رابط کاربری

**قدم‌ها:**

1. طراحی پنجره اصلی با استفاده از `QMainWindow`
    
2. اضافه کردن `QListWidget` برای نمایش محصولات
    
3. اضافه کردن `QTableWidget` برای نمایش سبد خرید
    
4. طراحی دکمه‌های عملیاتی (اضافه، حذف، پرداخت)
    

**خروجی:** یک رابط کاربری ابتدایی با بخش‌های مختلف.

### مرحله ۳: پیاده‌سازی مدل داده

**قدم‌ها:**

1. ایجاد کلاس `Product` با ویژگی‌های: نام، قیمت، توضیحات
    
2. استفاده از `dataclass` برای سادگی
    
3. ذخیره و بازیابی داده‌ها از فایل JSON
    

**کد نمونه:**

```python

from dataclasses import dataclass
import json
@dataclass
class Product:
    name: str
    price: float
    description: str = ""
```


### مرحله ۴: اتصال منطق به UI

**قدم‌ها:**

1. اتصال دکمه‌ها به توابع مربوطه
    
2. پیاده‌سازی منطق اضافه کردن به سبد خرید
    
3. پیاده‌سازی محاسبه مجموع قیمت
    
4. پیاده‌سازی ثبت سفارش
    

**خروجی:** برنامه‌ای که کاربر می‌تواند با آن کار کند.

### مرحله ۵: بهبود و تست

**قدم‌ها:**

1. اضافه کردن اعتبارسنجی ورودی‌ها
    
2. مدیریت خطاها (مثلاً اضافه کردن محصول تکراری)
    
3. نوشتن تست‌های واحد برای توابع اصلی
    
4. بهینه‌سازی کد
    

### مرحله ۶: ساخت فایل اجرایی

**قدم‌ها:**

1. نصب PyInstaller: `pip install pyinstaller`
    
2. تست ساخت فایل اجرایی برای ویندوز
    
3. تست ساخت فایل اجرایی برای لینوکس
    
4. مستندسازی مراحل ساخت در README
